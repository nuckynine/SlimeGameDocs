24 /* Rebind system depends on Unity's new input system. Most of the logic is designed around getting the bindings associated with each action, and displaying
25 * them correctly.
26 *
27 * Biggest Gotcha - the InputSettings inputActions refers to the object in the inspector. This can be read, but can't be modified (as far as I know). The
28 * generated C# class is where the modifications occur.
29 * Actions are read from the object, and used to manipulate the c# class.
30 * Design inspired by https://www.youtube.com/watch?v=TD0R5x0yL0Y
31 *
32 * ActionEntry is a prefab spawned in order to display each action, and its associated bindings as well as the controls to edit them.
33 */
